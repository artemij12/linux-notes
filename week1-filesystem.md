ls -a просмотр всех файлов в директории
ls -la просмотр всех файлов в директории с правами
ls просмотр файлов и папок в директории, доступна сортировка файлов по времени изменения и так далее.
pwd просмотр текущего местонахождения в файловой системе
cd для перемещения по файловой системе для поднятия на урвоень выше используй cd .. 
для возвращения в корень используй cd / для перехода в каталог текущего ползователья использхуй cd ~
mkdir создаем директории они же папки одной командой можно создать множество папок 
и с помощью -m присвоить режим файлазцв
mkdir -p создание цепочки папок для создания нескольких папок использовать {} если файлы однородные, то можно использовать следующую структуру:
mkdir -p ~/название1/folder{1..5}
mv - перемещение файлов или папок ВАЖНО: первы аргумент что перемещаем второй куда перемещаем
cp - копирование что копируем и куда копируем
rm - удаление -d- пустые директории -r рекурсивное удаление, удалим все что находится в папке включая сам папку
touch - создание пустых файлов и изменение временных меток фацйлов к которым обращаемся
touch в основном работает с двумя временными метками: atime — время последнего доступа к файлу, и mtime — время последнего изменения содержимого файла.
ls -lu показывает atime, а обычный ls -l показывает mtime.​
В Linux также есть ctime, которое связано с изменением метаданных файла, например прав доступа или других свойств

less - вывод для постарничного просмотра
clear очистка окна терминала

find предназначена для поиска файлов или директорий
шаблон find [директория поиска (если не указать используется текущая рекурсивно)] [параметры поиска] поисковый запрос
параметры поиска: 
type - тип файлы или директории f,d
-name по имени регистрозависимый поиск, если нужен не регистрозависимый использовать -iname так же можно искать файлы допустим по нескольим расширеням тогда нужно использовать склейку find -name "*.txt" -o -name "*.cpp" 
* работает во все стороны
-mtime время модификации за последнее время указывать напрямую число если интересует большее время назад то ставится + если меньшее то -7
-atime время доступа к файлу 
-user указывать имя пользователя который создал файл
так же можно склеивать несколько команд например  с использованием параметра -exec
-exec rm и далее то что нам нужно удалить 
шаблон команды find -name ".txt" -exec rm {} +
или другой вариант find -name ".txt" -exec rm {} \;
-perm поиск по правам доступа пользователя u+rwx
-size поиск по размеру + более какого-либо размера

grep используется для поиска совпадений в файлах
-n выводи совпавшую строку с ее номер в файле в котором производился поиск
-E поиск с регулярный выражением или же можно просто в строке поиска указать регулярное выражение
самое частое использование ^ для поиска в начале строки и $ для поиска в конце строки
обычно используется связка find  и grep для поиска нужного файла или файлов а затем для поиска внутри самих файлов, однако если нужно что-то найти внутри всех файлов в каталоге, то grep обладает параметром рекурсивного поиска -r и тогда будут просмотрено все содержимое всех файлов внутри каталога из которого выполнятеся команда
tail используется для вывода последних 10 строк файла
tail -n [количество последних строк] указывается путь до файла
tail -f вывод последних 10 строк с их текущим обновлением удобно для просмотра постоянно изменяющхся логов файлакомбинируем с grep и получаем поиску нужного в последних строках файла
удобно использовать
tail -f [путь до файла]|grep "шаблон поиска" поиск в постоянно обновляемых сроках и вывод этогору
head поиск в первых строках файла
head -n [количество строк] поиск в n первых строках файла
аналогично можно использовать с grep
head -n [путь до ффайла]| grep "шаблон поиска"
совмещаем поиск find tail/head и grep
find -name [шаблон поиска] -exec tail/head -n/-f количество строк {} + | grep "шаблон поиска"
wc команда для подсчета количества символов, байт внутри файла количества слов в файле, а также количества строк
sort используется для сортировки в алфавином порядке
sort -r реверсивный порядок
sort -n сортировка по числам
sort -u выводит только уникальные строки в отсортированном порядке
